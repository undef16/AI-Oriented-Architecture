### TIP-003: Repository Search Is the Real Bottleneck in AI Coding Agents

Status: Recommended for large-scale agentic systems.

### Context

Most teams assume AI coding agents spend most of their time generating code.

They don't.

In autonomous software engineering systems, 30%-40% of total execution time is spent on:

* repository search;
* dependency traversal;
* code reading;
* architectural planning.

Not coding.

Modern agents burn millions of tokens navigating repositories before writing a meaningful patch.

### What Happened

As repositories scale, AI agents fall into expensive search loops:

* grepping files;
* re-reading dependency chains;
* reconstructing architectural relationships;
* replaying accumulated context across turns.

The actual code fix is tiny.

The real cost is repository navigation.

Traditional RAG systems only partially solve this problem.

Why?

Because source code is not documentation.

It is a dependency graph.

### Why Traditional RAG Fails

Most RAG systems retrieve semantically similar text chunks.

But AI coding agents need:

* caller/callee relationships;
* blast-radius analysis;
* dependency tracing;
* architectural impact mapping;
* test relationship awareness.

Semantic similarity alone cannot provide this reliably.

### Why Code Graph Matters?

[Code Review Graph](https://code-review-graph.com/) replaces blind repository traversal with structural graph navigation — dramatically reducing token consumption during search, reading, and planning phases.

This directly attacks the most expensive phase of autonomous coding agents: repository search and planning.

Instead of pushing massive repositories into the context window, it exposes architectural relationships directly through graph queries and MCP tooling.

Results:

* 38x-528x token reduction
* Faster planning loops
* Smaller context windows
* Reduced retry cycles
* More accurate code modification targeting
* Targeted test execution instead of full-suite runs

The key advantage:

Traditional RAG asks:
What text looks similar?

code-review-graph asks:
What code is structurally connected?

At repository scale, that difference becomes economically critical.

### Recommendations

For AI engineering teams:

* Stop treating repositories as flat text corpora
* Optimize repository traversal before upgrading models
* Use structural retrieval for autonomous coding agents
* Minimize context carry-over between execution loops
* Treat token efficiency as a systems architecture problem

The biggest cost in AI coding systems is no longer generation.

It is navigation.
